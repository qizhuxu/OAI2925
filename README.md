# OAI2925

OpenAI 自动注册工具 - 专为 2925.com 邮箱优化的 Chrome 扩展

[![Version](https://img.shields.io/badge/version-1.0.0-blue.svg)](https://github.com)
[![License](https://img.shields.io/badge/license-MIT-green.svg)](LICENSE)
[![Chrome](https://img.shields.io/badge/chrome-extension-orange.svg)](https://developer.chrome.com/docs/extensions/)

## 概述

OAI2925 是一个自动化 Chrome 扩展，用于简化 OpenAI 账号注册流程。该工具专为 2925.com 邮箱服务优化，支持批量自动注册、智能验证码获取和完整的注册流程自动化。

## 核心特性

### 🎯 专注 2925 邮箱
- 移除其他邮箱提供商支持，专注优化 2925.com
- 自动生成格式化邮箱地址
- 智能收件人匹配，确保验证码准确性

### 🤖 全自动流程
- 支持 1-50 次批量自动注册
- 9 步完整注册流程自动化
- 智能错误检测和自动重试

### 🔒 安全可靠
- 手机验证自动检测
- 验证码精确匹配机制
- 邮件删除验证，防止重复使用

### ⚙️ 灵活配置
- CPA 模式 / 本地模式双模式支持
- 可选的 VPS 面板集成
- 本地文件保存功能

## 安装指南

### 前置要求

#### 基础环境
- Chrome/Edge 浏览器（版本 88+）
- 稳定的网络连接

#### 必须准备
1. **2925 邮箱**
   - 访问 [https://2925.com](https://2925.com) 并登录
   - 保持登录状态（建议不要关闭标签页）

2. **代理配置**（强烈推荐）
   - 推荐：[Webshare.io](https://webshare.io)（10 个免费 IP，风险度低）
   - 安装：[Webshare Proxy 扩展](https://chromewebstore.google.com/detail/webshare/bdokeillmfmaogjpficejjcjekcflkdh)
   - 作用：大幅降低触发手机验证（add-phone）的概率

#### 可选准备（根据模式选择）
- **CPA 模式**：登录 CPA 管理面板，保持登录状态
- **本地模式**：准备保存文件夹（如 `D:\OpenAI_Accounts`）

### 安装步骤

1. **下载扩展**
   - 克隆或下载本仓库
   - 解压到本地目录

2. **加载扩展**
   ```
   1. 打开 Chrome 浏览器
   2. 访问 chrome://extensions/
   3. 开启右上角的"开发者模式"
   4. 点击"加载已解压的扩展程序"
   5. 选择 OAI2925 文件夹
   ```

3. **启用权限**
   - 点击扩展详情
   - 启用"在无痕模式下启用"（如果使用无痕模式）
   - 确认所有权限已授予

## 使用指南

### 首次配置

1. **打开侧边栏**
   - 点击浏览器工具栏的扩展图标
   - 侧边栏将在右侧打开

2. **配置邮箱前缀**（必填）
   - 在"邮箱前缀"输入框填写自定义前缀
   - 示例：输入 `myprefix`
   - 生成格式：`myprefix_xxxx@2925.com`
   - 随机后缀为 4 位字符（字母+数字）

3. **选择运行模式**

   #### 本地模式（推荐个人使用）
   - **CPA 接口地址**：留空
   - **保存到本地**：勾选此选项
   - **本地保存路径**：点击"选择文件夹"选择保存位置
   - 认证文件将保存为 JSON 格式到指定文件夹

   #### CPA 模式（适合团队管理）
   - **CPA 接口地址**：填写 VPS 面板地址（如 `https://your-vps.com/management.html#/oauth`）
   - **CPA 管理密钥**：填写管理密钥（用于下载认证文件）
   - **保存到本地**：（可选）勾选后同时保存到本地
   - 认证文件将上传到 CPA 并可选保存到本地

4. **其他设置**
   - **运行次数**：设置批量注册的次数（1-50）
   - **无痕模式**：勾选后在无痕窗口中运行（需先启用扩展的无痕权限）

### 运行模式

#### 手动模式
逐步执行每个步骤，适合调试和学习：

1. 点击侧边栏中的步骤按钮（Step 1 - Step 9）
2. 观察每步的执行结果
3. 查看日志输出了解详细信息

#### 自动模式
批量自动注册，适合生产使用：

1. 设置运行次数（建议 1-10 次）
2. 点击"自动执行"按钮
3. 扩展将自动完成所有步骤
4. 在"账号"标签查看生成的账号
5. 在"日志"标签监控执行状态

### 注册流程说明

扩展支持两种运行模式，流程略有不同：

#### 本地模式流程

| 步骤 | 名称 | 说明 |
|------|------|------|
| Step 1 | 生成 OAuth 链接 | 本地生成 OAuth 授权链接（PKCE 方式） |
| Step 2 | 打开注册页面 | 导航到 OpenAI 注册页面 |
| Step 3 | 填写邮箱密码 | 自动生成并填写邮箱和随机密码 |
| Step 4 | 获取注册验证码 | 从 2925 邮箱获取验证码并填写 |
| Step 5 | 填写个人信息 | 自动生成随机姓名和生日 |
| Step 6 | OAuth 登录 | 使用生成的账号进行 OAuth 登录 |
| Step 7 | 获取登录验证码 | 从 2925 邮箱获取登录验证码 |
| Step 8 | 完成 OAuth | 点击授权按钮，检测手机验证，获取 code |
| Step 9 | 交换 Token | 使用 code 交换 access_token 和 refresh_token |

**本地模式特点**：
- ✅ 无需外部服务
- ✅ 完全本地化操作
- ✅ 自动保存认证文件到本地
- ⚠️ 需要手动处理 localhost 回调

#### CPA 模式流程

| 步骤 | 名称 | 说明 |
|------|------|------|
| Step 1 | 获取 OAuth 链接 | 从 CPA 接口获取预配置的 OAuth 链接 |
| Step 2 | 打开注册页面 | 导航到 OpenAI 注册页面 |
| Step 3 | 填写邮箱密码 | 自动生成并填写邮箱和随机密码 |
| Step 4 | 获取注册验证码 | 从 2925 邮箱获取验证码并填写 |
| Step 5 | 填写个人信息 | 自动生成随机姓名和生日 |
| Step 6 | OAuth 登录 | 使用生成的账号进行 OAuth 登录 |
| Step 7 | 获取登录验证码 | 从 2925 邮箱获取登录验证码 |
| Step 8 | 完成 OAuth | 点击授权按钮，检测手机验证，等待回调 |
| Step 9 | VPS 验证 | 通过 VPS 面板提交回调 URL 并下载认证文件 |

**CPA 模式特点**：
- ✅ 集成 VPS 面板
- ✅ 自动下载认证文件
- ✅ 支持 CPA 管理密钥
- ⚠️ 需要配置 CPA 接口地址

#### 模式对比

| 特性 | 本地模式 | CPA 模式 |
|------|---------|---------|
| OAuth 链接来源 | 本地生成 | CPA 接口 |
| Step 9 操作 | Token 交换 | VPS 验证 |
| 认证文件保存 | 本地文件夹 | CPA + 本地 |
| 外部依赖 | 无 | 需要 CPA 服务 |
| 适用场景 | 个人使用 | 团队/批量管理 |

## 邮箱生成规则

### 格式
```
{emailPrefix}_{random4}@2925.com
```

### 组成部分
- **emailPrefix**：用户设置的前缀（建议 3-20 个字符）
- **random4**：4 位随机字符
- **字符集**：`abcdefghjkmnpqrstuvwxyz23456789`（排除易混淆字符 `0`, `1`, `i`, `l`, `o`）

### 示例
```
前缀: myapp
生成: myapp_k7m3@2925.com
      myapp_3ma9@2925.com
      myapp_a9zx@2925.com
```

## 高级功能

### 手机验证检测
扩展会自动检测 OpenAI 的手机验证要求：
- step8检测到 `/add-phone` 页面时自动跳过
- 记录失败原因到日志
- 自动开始下一次注册流程
- 不影响后续注册

### 智能验证码匹配
确保获取正确的验证码：
- **收件人匹配**：从 bounce 地址提取收件人信息
- **时间容差**：65 秒时间窗口，适配分钟级时间戳
- **格式验证**：严格匹配验证码格式
- **去重机制**：防止重复使用同一验证码

### 邮件删除验证
确保邮件正确删除：
- 3 次重试机制
- DOM 元素验证
- 防止验证码重复使用
- 保持邮箱清洁

### 错误处理
- 全局错误监测器（自动点击重试按钮）
- 人工介入机制（遇到问题时暂停等待）
- 详细的错误日志记录
- 自动重试失败步骤

## 技术架构

### 技术栈
- **Manifest V3**：最新的 Chrome 扩展标准
- **Service Worker**：后台任务处理
- **Content Scripts**：页面交互和自动化
- **Chrome Debugger API**：精确的 UI 交互
- **File System Access API**：本地文件保存

```

│   ├── signup-page.js     # 注册页面自动化
```

## 常见问题

### Q: 为什么必须登录 2925 邮箱？
A: 扩展需要访问 2925.com 获取验证码邮件。首次使用前请手动登录 2925.com。

### Q: 批量注册建议多少次？
A: 建议每次 5-10 次。过多可能触发 OpenAI 的风控机制。

### Q: 遇到手机验证怎么办？
A: 扩展会自动检测并跳过，继续下一次注册。如果频繁遇到手机验证：

**立即措施**：
1. 更换代理 IP（最有效）
2. 等待 10-30 分钟后再试
3. 降低注册频率

**预防措施**：
1. 使用 [Webshare.io](https://webshare.io) 等高质量代理
2. 每个 IP 只注册 1-3 个账号
3. 避免使用数据中心 IP
4. 单次运行不超过 10 个账号

**代理配置**：
- 安装 [Webshare Proxy 扩展](https://chromewebstore.google.com/detail/webshare/bdokeillmfmaogjpficejjcjekcflkdh)
- 配置住宅代理或高质量数据中心代理
- 定期轮换 IP 地址

### Q: 如何查看生成的账号？
A: 点击侧边栏的"账号"标签，可以查看所有生成的账号，并支持导出为 JSON 文件。

### Q: CPA 模式和本地模式有什么区别？
A: 

**本地模式**：
- ✅ 无需外部服务，完全本地化
- ✅ 自动生成 OAuth 链接（PKCE 方式）
- ✅ 直接交换 Token 并保存到本地
- ✅ 适合个人使用
- ⚠️ 需要处理 localhost 回调

**CPA 模式**：
- ✅ 集成 VPS 面板管理
- ✅ 从 CPA 接口获取 OAuth 链接
- ✅ 通过 VPS 验证并自动下载认证文件
- ✅ 支持团队协作和批量管理
- ⚠️ 需要配置 CPA 服务地址和管理密钥

**如何选择**：
- 个人使用 → 本地模式
- 团队管理 → CPA 模式
- 不确定 → 先用本地模式

### Q: 为什么 Step 8 会失败？
A: Step 8 使用 Chrome Debugger API 点击按钮。如果失败，请确保：
1. 已授予 debugger 权限
2. 没有其他调试工具占用
3. 页面已完全加载

## 故障排除

### 扩展无法加载
- 检查 Chrome 版本是否 88+
- 确认开发者模式已开启
- 查看控制台是否有错误信息

### 无法获取验证码
- 确认已登录 2925.com
- 检查邮箱前缀是否正确设置
- 查看 2925 邮箱是否收到邮件

### Step 8 一直失败
- 关闭其他调试工具
- 重新加载扩展
- 检查 debugger 权限是否授予

### 日志查看
- 点击侧边栏"日志"标签查看详细日志
- 按颜色区分日志级别（info/ok/warn/error）
- 使用"清空日志"按钮清理旧日志
